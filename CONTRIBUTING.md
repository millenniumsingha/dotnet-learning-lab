# Contributing to DotNet Learning Lab

## Workflow

1. **Create an Issue:** Describe the change or bug.
2. **Create a Branch:** specific to the issue (e.g., `feat/my-feature-123`) from `phase/v1`.
3. **Implement Changes:** Follow coding standards.
4. **Verify Locally:** Build and test.
5. **Create Pull Request:** Target `phase/v1`. Ensure CI passes.
6. **Merge:** Squash or Rebase/Merge as per repository settings.

## Documentation

- See [ARCHITECTURE.md](ARCHITECTURE.md) for solution structure.
- See the project's [GitHub Issues](https://github.com/millenniumsingha/dotnet-learning-lab/issues) for the roadmap.

## Style Guide

- Follow C# coding conventions.
- Use the provided `.editorconfig`.
- Ensure all new public APIs are documented.
