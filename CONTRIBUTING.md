# Contributing to DotNet Learning Lab

## Workflow
1. Create an issue describing the change.
2. Create a branch from `phase/v1` (e.g., `feat/my-feature-123`).
3. Implement changes.
4. Verify tests and build locally.
5. Create a Pull Request targeting `phase/v1`.
6. Ensure CI passes.

## Style Guide
- Follow C# coding conventions.
- Use the provided `.editorconfig`.
